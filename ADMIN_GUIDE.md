# Инструкция по использованию админ-панели

## Доступ к админ-панели

### Вход в систему

1. Откройте в браузере: `http://localhost:3000/admin/login`
2. Введите пароль администратора (по умолчанию: `admin123`)
3. Нажмите кнопку "Войти"

**Примечание:** Пароль хранится в файле `.env` в переменной `ADMIN_PASSWORD`. Для изменения пароля отредактируйте этот файл.

### Выход из системы

- Нажмите кнопку "Выйти" в боковом меню

---

## Главная страница (Dashboard)

После входа вы попадаете на главную страницу админ-панели по адресу `/admin`.

### Что отображается:

- **Статистика:**
  - Общее количество категорий
  - Общее количество слов
  
- **Последние категории:**
  - Список из 5 последних добавленных категорий
  - Для каждой категории показано количество слов

### Навигация:

В левом боковом меню доступны разделы:
- **Dashboard** - главная страница
- **Категории** - управление категориями слов
- **Слова** - управление словами

---

## Управление категориями

Откройте раздел "Категории" в меню или перейдите по адресу `/admin/categories`.

### Просмотр категорий

Отображается таблица со всеми категориями:
- **ID** - уникальный номер категории
- **Название** - название категории (например, "Животные")
- **Slug** - URL-дружественное имя (например, "animals")
- **Слов** - количество слов в категории
- **Действия** - кнопки управления

### Создание новой категории

1. Нажмите кнопку **"Добавить категорию"** в правом верхнем углу
2. В открывшемся окне введите название категории
3. Нажмите **"Создать"**

**Примечание:** Slug генерируется автоматически из названия (пробелы заменяются на дефисы, убираются спецсимволы).

**Пример:**
- Название: `Животные и птицы`
- Slug: `zhivotnye-i-pticy`

### Редактирование категории

1. Найдите нужную категорию в таблице
2. Нажмите кнопку **"Изменить"** в столбце "Действия"
3. Измените название
4. Нажмите **"Сохранить"**

### Удаление категории

1. Найдите нужную категорию в таблице
2. Нажмите кнопку **"Удалить"** в столбце "Действия"
3. Подтвердите удаление в диалоговом окне

**⚠️ Внимание:** При удалении категории все связи со словами также удаляются, но сами слова остаются в базе данных.

## Управление словами

Откройте раздел "Слова" в меню или перейдите по адресу `/admin/words`.

### Просмотр слов

Отображается таблица со всеми словами:
- **Чекбокс** - для массового выбора
- **ID** - уникальный номер слова
- **Текст** - текст слова
- **Категории** - список категорий, к которым относится слово (в виде цветных бейджей)
- **Действия** - кнопки управления

### Поиск слов

1. Введите текст в поле поиска вверху страницы
2. Нажмите кнопку **"Найти"**
3. Таблица обновится и покажет только слова, содержащие введенный текст

**Примечание:** Поиск работает по частичному совпадению (например, поиск "кот" найдет "кот", "котенок", "котлета").

### Пагинация

- На одной странице отображается **50 слов**
- Используйте кнопки **"Назад"** и **"Вперед"** для навигации между страницами
- Текущая страница и общее количество страниц отображаются в центре

### Создание нового слова

1. Нажмите кнопку **"Добавить слово"** в правом верхнем углу
2. В открывшемся окне:
   - Введите текст слова
   - Выберите одну или несколько категорий (обязательно!)
3. Нажмите **"Создать"**

**Примечание:** Слово должно относиться хотя бы к одной категории.

**Пример:**
- Текст: `Собака`
- Категории: ✅ Животные, ✅ Домашние питомцы

### Редактирование слова

1. Найдите нужное слово в таблице
2. Нажмите кнопку **"Изменить"** в столбце "Действия"
3. Измените текст или категории
4. Нажмите **"Сохранить"**

### Удаление одного слова

1. Найдите нужное слово в таблице
2. Нажмите кнопку **"Удалить"** в столбце "Действия"
3. Подтвердите удаление в диалоговом окне

### Массовое удаление слов

1. Отметьте чекбоксы у нужных слов (или используйте чекбокс в заголовке таблицы для выбора всех)
2. Нажмите кнопку **"Удалить выбранные (N)"**, которая появится вверху таблицы
3. Подтвердите удаление в диалоговом окне

**Примечание:** Можно удалить до 50 слов за раз (все слова на текущей странице).

## Технические особенности

### Автоматическая синхронизация

После любых изменений (создание, редактирование, удаление) данные автоматически обновляются:
- В админ-панели
- На странице настроек игры (`/game/settings`)
- В активных играх

Не требуется перезагружать страницу или очищать кеш.

### Валидация данных

**Категории:**
- Название обязательно
- Название должно быть уникальным
- Slug генерируется автоматически и должен быть уникальным

**Слова:**
- Текст обязателен
- Текст должен быть уникальным
- Должна быть выбрана хотя бы одна категория

### Обработка ошибок

При возникновении ошибок (например, попытка создать дубликат) отображается сообщение об ошибке красным цветом под формой.

**Примеры ошибок:**
- "Категория с таким названием уже существует"
- "Слово уже существует"
- "Выберите хотя бы одну категорию"

## Безопасность

### Защита админ-панели

- Все страницы админ-панели (кроме `/admin/login`) защищены паролем
- При попытке доступа без авторизации происходит автоматический редирект на страницу входа
- Токен аутентификации хранится в защищенных HTTP-only cookies
- Токен действителен 7 дней

### Рекомендации по безопасности

1. **Измените пароль по умолчанию:**
   - Откройте файл `.env`
   - Измените значение `ADMIN_PASSWORD=admin123` на надежный пароль
   - Перезапустите сервер

2. **Не коммитьте файл `.env` в Git:**
   - Файл `.env` уже добавлен в `.gitignore`
   - Никогда не публикуйте пароль в открытом доступе

3. **Для production:**
   - Используйте сложный пароль (минимум 16 символов)
   - Рассмотрите возможность добавления rate limiting для защиты от брутфорса

## Часто задаваемые вопросы

**Q: Можно ли добавить слово без категории?**  
A: Нет, каждое слово должно относиться хотя бы к одной категории.

**Q: Что произойдет, если удалить категорию, в которой есть слова?**  
A: Категория удалится, связи со словами удалятся, но сами слова останутся в базе данных.

**Q: Как изменить пароль администратора?**  
A: Откройте файл `.env`, измените значение `ADMIN_PASSWORD`, перезапустите сервер.

**Q: Можно ли импортировать слова из файла?**  
A: В текущей версии импорт не реализован. Эта функция запланирована для будущих обновлений.

**Q: Сколько слов можно добавить?**  
A: Ограничений нет. База данных поддерживает неограниченное количество слов.

**Q: Как посмотреть историю изменений?**  
A: В текущей версии история не ведется. Эта функция запланирована для будущих обновлений.
